Overview

Infernal ("INFERence of RNA ALignment") is for searching DNA sequence databases for RNA structure and sequence similarities. It is an implementation of a special case of profile stochastic context-free grammars called covariance models (CMs). A CM is like a sequence profile, but it scores a combination of sequence consensus and RNA secondary structure consensus, so in many cases, it is more capable of identifying RNA homologs that conserve their secondary structure more than their primary sequence.

The latest release of Infernal is 1.1rc1 [27 Jun 2012].

Note on the Rfam transition to Infernal 1.1

The current Rfam 10.1 database is built on Infernal 1.0. This means that CM files from Rfam cannot be used directly with Infernal 1.1. If you'd like to use Infernal 1.1 for Rfam 10.1 work, you have three options.

First, you can use the cmconvert program in Infernal 1.1 to convert the 1.0 files from Rfam. Second, you can download all converted Rfam 10.1 CMs in a single gzipped CM file in 1.1 format, and use that. With these first two options, the models will be parameterized identically to the 1.0 built Rfam 10.1 models available from the Rfam website, so identical hits using the 1.0 and 1.1 files will have identical bit scores. You'll want to use one of these two options if you want to reproduce scores obtained with the Infernal 1.0 Rfam files.

The third option is to download a file with newly built and parameterized Rfam 10.1 CMs created by Infernal 1.1's cmbuild and cmcalibrate . These models were built from the Rfam 10.1 seed alignments but using Infernal 1.1's parameterization methods. With this third option, the models will have slightly different parameters than the Infernal 1.0 Rfam 10.1 CM files, so the same hit will likely get a different score and E-value using these models compared with using the Infernal 1.0 Rfam 10.1 models. However, our benchmarking suggests that the new parameterization makes the models slightly more sensitive, so you may want to choose this option if you're not worried about reproducing Rfam 10.1 bit scores.

Alternatively, you can continue using the previous Infernal 1.0.2 release for Rfam work until Rfam makes the transition to Infernal 1.1.

Further reference:

  • Several improvements in Infernal 1.0, including techniques used to improve speed performance, are described in Eric Nawrocki's 2009 Ph.D. thesis.
  • The trCYK local alignment algorithm is described in (Kolbe and Eddy, 2009).
  • The QDB query-dependent banded dynamic programming algorithm used to increase search speed is described in (Nawrocki and Eddy, 2007), along with mixture Dirichlet priors and entropy weighting.
  • The Rfam database of RNA families is based on the Infernal software (Griffiths-Jones, 2003; Griffiths-Jones, 2005) .
  • The RSEARCH software for searching with single RNA structures (instead of with profiles built from multiple alignments) is based on Infernal's alignment engines (Klein and Eddy, 2003). RSEARCH also included MPI parallelization and E-value calculations, features that were merged back into Infernal in Infernal 1.0.
  • The memory-efficient divide-and-conquer CYK alignment algorithm is described in (Eddy, 2002). A new implementation, Infernal, replaced COVE.
  • A chapter of Biological Sequence Analysis (Durbin et al., 1998) describes CMs in more detail.
  • The tRNA identification program tRNAscan-SE is based on a tRNA CM and COVE (Lowe and Eddy, 1997).
  • Covariance models were originally described in (Eddy and Durbin, 1994). They were first implemented in the COVE software package.
